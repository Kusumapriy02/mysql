mysql> -- the process of combining all related operations into a single unit and excuting them according to rule "either all operation should be succed or none should be succed is called transaction.
mysql> -- A transaction is considered a single unit of work and follows the princple ,"either all or none"
mysql> -- example: funds transfer
mysql> -- consider a money transfer process between two accounts      1. debit amount from the sender's account
mysql> -- 2. credit amount into the receiver's account
mysql> -- both operation should be excuted as a single transaction.
mysql> -- if the amount is debited from senders account succesfully but, crediting the recovers account fails. it may lead to the data inconsistency problems.
mysql> -- there are two types of transactions such as local transactions and global transaction
mysql> -- local transaction : in this all operation of a transaction s are excuted in same data base 
mysql> -- example : funds transfer from one account to another account where both accounts belong to same bank.
mysql> -- global transaction: in global transaction, operation of a transaction are excuted on different databases.
-example: funds transfer from one account to another account belongs to different bank.

    -> A(atomicity): all or nothing
You withdraw ₹5,000. The amount is deducted from your account only if the cash is dispensed. If the ATM fails to dispense cash, the deduction is rolled back.
    -> C(consistency): the database remains in valid stateIf your balance is ₹10,000 and you withdraw ₹5,000, the new balance becomes ₹5,000. The database never shows an invalid balance.
    -> I(isolation):Transactions dont interface eachother
If you and another person access the same account simultaneously, one transaction completes before the other affects the balance, preventing incorrect results.
    -> D(durability):changes are permanent.
After the ATM confirms the withdrawal, the updated balance remains saved even if there is a power failure or the bank server restarts.

mysql> -- locking: it is a mechanism used by the databse to control concurrent access data.
mysql> -- it ensure that users access the same data simultaneously the database maintaince consistency,integrity,and prevent conflicts.
mysql> -- shared lock
mysql> -- exclusive lock
mysql> -- shared lock: A shared lock allows a transaction to read data,multiple transaction can hold a shared lock on the same row. no transaction can modify the row until all shared locks are released.
mysql> --characterstic: read only,multiple users can read simultaneously,blocks update and delete
    -> exclusive lock: An exclusive lock allows a transaction to modify data.
    -> while an exclusive lock exists:
    -> No other transaction can read using a locking read or modify row
    -> only the transaction holding the lock can update /delete the data
    -> characterstics:
    -> used for insert,update,delete
    -> only one exclusive lock at a time
    -> blocks both reads and writes
    -> Intent lock:
    -> An intent lock is a table-level lock that indicates a transaction intends to place row level locks.
    -> it helps Mysql quickly determine whether a table-level lock conflicts with exsiting row locks.
    -> users do not usually create intent lock manually .INNODB manages them automatically
    -> real time example:
    -> shared lock: many stuents can read the same book at the same time
    -> exclusive lock:
    -> one student borrows the book , so nobody else can read or borrow it until it is returned.
    -> intent lock: The librarain marks that someone is about to read or borrow books from a shelf,
    -> helping coordinate access without checking everybook individually.
